TripMates API

Trips Routs

Method URL Body Description
GET domain/animals - Get a list of all the animals. It will present 10 results per page
GET domain/animals?perPage={x}&page={y} - Get a list of all the animals. It will present x results per page and will take you to page y
GET domain/animals/search?s={name or info} - Get a list of the animals by name or info. It will present 10 results per page
GET domain/animals/search?s={name_or_info}&perPage={x}&page={y} - Get a list of the animals by name or info. It will present x results per page and will take you to page y
GET domain/animals/category/{category} - Get a list of the animals by category. It will present 10 results per page
GET domain/animals/category/{category}?perPage={x}&page={y} - Get a list of the animals by category. It will present x results per page and will take you to page y
GET domain/animals/prices?min={x}&max={y} - Shows only the animals that their price is less/equal x and greater/equal y
GET domain/animals/prices?min={x}&max={y}&perPage={z}&page={w} - Shows only the animals that their price is less/equal x and greater/equal y. It will present z results per page and will take you to page w
POST domain/animals {
"name": "joni",
"info": "love kids and cats",
"category": "dog",
"img_url": "https://www.pexels.com/photo/dog/",
"price": 350
} 		Add a new animal to the data base. You need to send a token in the `Headers` for this query.
(x-api-key)
PUT domain/animals/{animal_Id} {
"name": "joni",
"info": "doesn't love kids and cats",
"category": "dog",
"img_url": "https://www.pexels.com/photo/dog/",
"price": 1200
} 		Update an animal in the data base. You need to send a token in the `Headers` for this query.
(x-api-key)
DELETE domain/animals/{animal_Id} - Delete an animal in the data base. You need to send a token in the `Headers` for this query.
(x-api-key)

Users Routs

Method URL Body Description
GET domain/users - Get a list of all the users. Only the admin can get this information. You need to send a token in the `Headers` for this query.
(x-api-key)
POST domain/users {
"name":"yehuda",
"email":"yehuda@gmail.com",
"password":"123456"
} 		Add a new user to the data base
POST domain/users/login {
"email":"yehuda@gmail.com",
"password":"123456"
} 		User login. If the email and password are correct the server will respond with a token
